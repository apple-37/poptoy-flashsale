# 在线盲盒与潮玩秒杀系统 - SRS

## 1. 项目介绍
本项目是一个面向高并发场景的纯后端电商微服务系统，主打“潮玩限量秒杀”与“盲盒在线抽赏”两大核心业务。系统摒弃了传统低效的轮询机制，采用 Redis 预扣减、RabbitMQ 异步排队、死信队列 (DLX) 处理超时，并结合 SSE (Server-Sent Events) 实现服务端异步处理结果的实时推送。

## 2. 功能介绍
*   **用户鉴权中心**：支持用户注册、密码加密存储、无状态 JWT 登录鉴权。
*   **商品与奖池展示**：支持高频查询秒杀商品列表、盲盒奖池详情（采用冷热数据分离方案）。
*   **高并发秒杀/抽赏**：支持瞬时海量请求，严格保证一人一单，绝对防止超卖。
*   **异步交易流转**：订单异步创建，提供实时的前端通知；支持订单支付（模拟），并对 15 分钟未支付的订单进行自动释放和库存回滚。

## 3. 功能需求 (Functional Requirements)
1.  **用户模块**：密码必须经过 Bcrypt 加密；登录发放带有过期时间的 JWT Token。
2.  **商品模块**：活动开始前，管理员（通过脚本模拟）将库存预热至 Redis。
3.  **交易模块**：
    *   接收下单请求时，通过 Lua 脚本在 Redis 中完成原子校验和扣减。
    *   扣减成功后，将下单 Task（附带用户信息、商品信息）投递至 RabbitMQ，并立即向客户端返回 202。
    *   异步 Worker 消费 Task 创建数据库订单，完成后通过事件机制通知前端。
4.  **超时控制**：Worker 创建订单后，将带有 15分钟 TTL 的消息投递至延迟队列。超时后消息流转至死信队列 (DLX)，由专门的消费者检查订单状态，若未支付则执行回滚。

## 4. 非功能需求 (Non-Functional Requirements)
*   **性能**：单节点核心秒杀接口 QPS 目标 > 3000，P99 响应时间 < 50ms。
*   **高可用**：核心抢购链路不强依赖 MySQL，MySQL 宕机或变慢不影响前端抢购排队。
*   **一致性**：保证 Redis 缓存库存与 MySQL 最终库存的最终一致性；保证状态机流转的严格一致性。
